# 克记忆库 LMC-5 闭环 v2 验收边界

## 目标

补齐记忆的数据生命周期和灾备闭环，让克更完整地保有、整理和联想自己的经历。

## 自由边界

- 不修改克现有的唤醒分类和读取配额。
- 不向对话 prompt 自动注入口吻、关系台词或行为示范。
- E 轴只做影子评分，不控制语气、态度或事实判断。
- X 轴叙事只写独立摘要表，不自动写成正式记忆。
- 自发浮现只提供可选缓存，不自动进入 prompt，不作为关系图扩展种子。
- 夜间任务不自动发布候选，不自动解决事实冲突，不自动批准风险关系边。

## 本版闭环

1. SQLite 快照、哈希校验、受控恢复和恢复前安全副本。
2. 正式记忆软删除；读取默认排除已删除内容。
3. dry-run、候选列表和巡逻预览保证零写入。
4. embedding 远程请求前脱敏，并提供不泄露密钥的健康统计。
5. E 轴 rules-v4：技术记忆默认不评分，关系/体验信号保持 shadow-only。
6. Y 轴只自动生成有证据的安全边；风险边进入独立审核队列。
7. Z 轴只报告多 current 事实，不自动裁决。
8. X 轴生成确定性的周/月叙事摘要，不调用模型。
9. M 轴按保护、时间、激活、重要度和体验信号做只读代谢审计。
10. 自发记忆缓存带 N 轮防重复；默认不自动消费。
11. 夜间闭环顺序：快照 -> 软处理短期过期 -> raw 分块 -> 候选 -> 安全 Y -> Z 审计 -> E shadow -> X 叙事 -> 自发缓存 -> M/边健康 -> 报告。
12. Git 灾备覆盖新增叙事和关系审核表；缓存和召回轨迹可重建，不作为永久备份。

## 上线顺序

1. 本地运行 `npm.cmd run test:lmc5`。
2. 推送 memory server，等待 Zeabur 完成部署。
3. 调用 `create_memory_snapshot(dry_run=false)` 建立真实库基线。
4. 调用 `run_lmc_night_maintenance(dry_run=true)` 检查计划。
5. 确认无异常后再运行一次 `dry_run=false`。
6. 更新 VPS `scripts/patrol.py`；保留旧链作为滚动部署回退。
7. 次日检查报告、候选、E 轴和坏边，不自动修复报告项。

## 本地验收

- `server/test_lmc5.mjs`：主链、零写入、候选/审核、关系、叙事、自发缓存和夜间闭环。
- `server/test_lmc_closure.mjs`：真实快照、故意改坏、计划恢复、模拟重启、哈希校验和内容恢复。
- `scripts/patrol.py`：Python 语法检查通过；新版单入口失败时回退旧链。
